# 儿童拍照识词绘本 App PRD

版本：v0.1  
日期：2026-06-05  
产品代号：毛毛英语绘本 / Felt Words  
平台：iOS 优先  
目标用户：3-8 岁儿童及其家长

## 1. 产品定位

本产品是一个面向儿童的英语启蒙 App。孩子通过拍照识别身边物体，学习对应英文单词，并把识别出的单词生成短篇连环画绘本和自动朗读小视频。

核心定位：

> 用真实世界里的物品，生成孩子自己的英文小绘本。

产品不做开放式 AI 聊天，不做复杂课程系统，不做社交社区。第一阶段聚焦低龄儿童最容易理解的学习闭环：

拍照识物 -> 学英文单词 -> 加入单词本 -> 生成 4 页故事绘本 -> 自动朗读播放 -> 复习巩固。

## 2. 背景与机会

传统儿童英语学习 App 多从课程、题库、卡片出发，孩子面对的是预置内容，和真实生活连接不强。本产品从孩子身边的真实物体出发，例如苹果、杯子、玩具车、书包、小猫，让学习发生在日常场景中。

差异点：

- 学习入口自然：孩子拍自己看到的东西，而不是被动刷题。
- 内容个性化：每次拍照都能生成属于孩子的小故事。
- 记忆链路更完整：实物图像、英文单词、发音、句子、绘本故事互相强化。
- 儿童体验更轻：单次只学习 1-3 个词，不造成信息压力。

## 3. 产品目标

### 3.1 MVP 目标

- 让孩子能完成一次完整体验：拍照 -> 识别单词 -> 听发音 -> 生成绘本 -> 自动播放。
- 让家长理解产品价值：真实物体变成英文绘本。
- 验证儿童是否愿意重复拍摄与收藏单词。

### 3.2 核心指标

- 首次识别完成率：用户首次进入后成功识别至少 1 个物体的比例。
- 绘本生成率：识别成功后点击生成绘本的比例。
- 绘本播放完成率：4 页绘本自动播放完成的比例。
- 单词本保存率：识别结果被保存到单词本的比例。
- 次日回访率：儿童/家长第二天再次打开 App 的比例。

### 3.3 非目标

MVP 不做以下内容：

- 开放式儿童聊天
- 社区分享与陌生人互动
- 排行榜、PK、强竞争机制
- 长篇故事创作
- 完整 AI 动画视频生成
- 复杂账号体系
- 多端同步

## 4. 用户画像

### 4.1 儿童用户

年龄：3-8 岁  
特点：

- 喜欢拍照、点按钮、听声音、看动画。
- 阅读能力弱，依赖图像和语音反馈。
- 注意力持续时间短，单次体验应控制在 1-3 分钟。
- 对可爱角色、收集、奖励反馈敏感。

核心诉求：

- 我拍了一个东西，它能告诉我这是什么。
- 我想听它怎么读。
- 我想看它变成一个小故事。

### 4.2 家长用户

特点：

- 关注学习效果、隐私、安全和屏幕时间。
- 希望孩子学到真实有用的词。
- 不希望 App 出现不适合儿童的内容。

核心诉求：

- 孩子真的在学英文，而不是只玩动画。
- 照片和数据可控。
- 内容安全，不吓人，不暴力，不引导消费。

## 5. 核心体验原则

1. 一屏只完成一个主要动作。
2. 儿童侧少文字、多图形、多语音。
3. 单次学习不超过 3 个新词。
4. 每个 AI 生成结果必须可解释、可撤回、可重新生成。
5. 毛毡元素只做温暖手作感表达，例如羊毛纹理、缝线、贴布、纽扣和柔软边缘，不做尖锐或复杂装饰。
6. 任何照片上传、保存、删除都必须让家长可理解、可控制。

## 6. 信息架构

底部主导航为 5 个入口：

- 首页
- 拍一拍
- 我的绘本
- 单词本
- 历史记录

家长设置为后续独立入口，不占用当前底部主导航。

### 6.1 拍一拍

功能：

- 打开相机
- 对准物体
- 拍照识别
- 查看识别结果
- 播放英文发音
- 生成绘本
- 加入单词本

### 6.2 我的绘本

功能：

- 查看已生成绘本
- 继续播放绘本
- 删除绘本
- 查看绘本来源单词
- 自动播放模式

### 6.3 单词本

功能：

- 查看已学习单词
- 单词发音
- 按主题筛选：食物、动物、玩具、家庭、自然、交通
- 标记已掌握
- 从单词生成新故事

### 6.4 家长

功能：

- 设置儿童年龄
- 设置识别语言辅助：中文提示开/关
- 设置是否允许上传照片识别
- 删除全部本地数据
- 查看隐私说明
- 设置每日使用时长提醒

### 6.5 历史记录

功能：

- 每次识别成功后自动保存，不要求用户先加入单词本。
- 按识别时间倒序展示图片、英文、中文与存入时间。
- 支持一键存入单词本。
- 支持直接生成绘本。
- 首页“开始拍照”右侧提供可横向滑动的历史记录入口。

## 7. 核心用户流程

### 7.1 首次进入流程

1. 欢迎页展示产品主张：拍身边的东西，变成英文小绘本。
2. 家长选择孩子年龄段：3-4 岁、5-6 岁、7-8 岁。
3. 请求相机权限。
4. 进入拍照页。

### 7.2 拍照识别流程

1. 孩子进入拍照页。
2. 页面出现取景框和提示文案：把物品放进框里。
3. 点击底部大拍照按钮。
4. App 显示照片预览和识别中状态。
5. 返回识别结果：主单词、备选单词、置信度。
6. 识别成功后立即自动存入历史记录，图片生成完成后更新同一条记录。
7. 孩子或家长确认正确单词。
8. 展示单词卡：英文、发音、中文辅助、简单例句。
9. 用户选择加入单词本或生成小故事。

### 7.3 绘本生成流程

1. 用户点击生成小故事。
2. 系统根据 1-3 个单词生成 4 页故事。
3. 每页包含一张插图、一句英文、一个重点词。
4. 用户进入绘本阅读页。
5. 默认自动朗读第一页。
6. 用户可手动下一页或切换自动播放。
7. 播放完成后显示小奖励和复习入口。

### 7.4 单词复习流程

1. 用户进入单词本。
2. 选择单词或主题。
3. 播放发音。
4. 进行轻量练习：看图选词、听音选图、单词配对。
5. 完成后更新掌握状态。

## 8. 功能需求

### 8.1 相机识别

优先级：P0

需求：

- 支持实时相机预览。
- 支持拍照。
- 支持从相册选择图片，入口弱化，放在拍照页右下角。
- 拍照后显示识别中状态。
- 识别失败时提供重新拍照按钮。
- 返回 1 个主识别词和最多 3 个备选词。

识别结果字段：

- word：英文单词
- displayNameZh：中文辅助解释
- confidence：置信度
- category：分类
- childFriendlyDefinition：儿童友好英文释义
- exampleSentence：简单英文例句
- safetyStatus：安全状态

验收标准：

- 从点击拍照到出现识别结果，理想状态不超过 5 秒。
- 识别结果必须至少包含英文单词和发音入口。
- 置信度过低时，不直接进入绘本生成，应引导用户确认或重拍。

### 8.2 单词学习卡

优先级：P0

需求：

- 展示英文单词，大字号。
- 展示中文辅助，小字号，可在家长设置关闭。
- 展示儿童友好例句。
- 支持播放英文发音。
- 支持加入单词本。
- 支持生成绘本。
- 支持选择备选识别结果。

验收标准：

- 儿童无需阅读说明即可通过图标理解播放、收藏、生成故事。
- 单词加入单词本后不能重复创建，应更新学习次数。

### 8.3 绘本生成

优先级：P0

需求：

- 支持基于 1 个单词生成 4 页故事。
- 支持基于 2-3 个单词生成 4-6 页故事。
- 每页只有一句英文，句子长度按年龄控制。
- 每页至少有 1 个重点词。
- 插图风格保持统一。
- 支持重新生成故事，但每天次数可限制。

年龄分层：

- 3-4 岁：每句 3-5 个英文单词。
- 5-6 岁：每句 5-8 个英文单词。
- 7-8 岁：每句 8-12 个英文单词。

验收标准：

- 故事不得出现暴力、惊吓、危险行为、成人内容。
- 故事语气温和，结尾积极。
- 插图不得生成真实儿童面部替换、夸张恐怖角色或危险场景。

### 8.4 绘本阅读器

优先级：P0

需求：

- 支持左右翻页。
- 支持自动播放。
- 支持英文朗读。
- 当前朗读单词高亮。
- 支持暂停、继续、重播当前页。
- 支持保存到我的绘本。

验收标准：

- 单页画面以插图为主，文字不遮挡主体。
- 自动播放节奏适合儿童，每页停留 4-6 秒。
- 视频感通过自动翻页、字幕、高亮、转场实现，不要求 MVP 生成真实视频文件。

### 8.5 我的绘本

优先级：P1

需求：

- 列表展示绘本封面、标题、生成日期、来源单词。
- 支持继续阅读。
- 支持删除。
- 支持按来源词筛选。

验收标准：

- 绘本列表首屏至少展示 3 个绘本卡片。
- 删除需要家长确认或长按确认，避免儿童误删。

### 8.6 单词本

优先级：P0

需求：

- 保存识别过的单词。
- 展示单词图片、英文、中文辅助、掌握状态。
- 支持播放发音。
- 支持按分类浏览。
- 支持从单词重新生成故事。

验收标准：

- 同一单词多次识别时合并记录，保留最近图片。
- 离线时仍可查看已保存单词。

### 8.7 练习小游戏

优先级：P1

MVP 可做轻量版本：

- 看图选词：展示图片，选择正确英文。
- 听音选图：播放单词，选择对应图片。
- 配对：英文单词和图片配对。

验收标准：

- 每轮 3 题以内。
- 错误反馈不使用负面语言，改用“再试一次”。

### 8.8 家长中心

优先级：P0

需求：

- 设置孩子年龄段。
- 查看和修改隐私设置。
- 删除数据。
- 控制是否保存原始照片。
- 控制是否允许云端识别。
- 查看生成内容说明。

验收标准：

- 家长中心入口不应吸引儿童频繁点击。
- 删除数据必须二次确认。

## 9. AI 与内容生成要求

### 9.1 物体识别 Prompt 要求

输入：

- 图片
- 孩子年龄段
- 语言：英文学习为主，中文辅助

输出 JSON：

```json
{
  "objects": [
    {
      "word": "apple",
      "displayNameZh": "苹果",
      "category": "food",
      "confidence": 0.94,
      "childFriendlyDefinition": "An apple is a sweet fruit.",
      "exampleSentence": "I see a red apple.",
      "safeForChild": true
    }
  ]
}
```

规则：

- 只返回儿童适合学习的普通物体。
- 遇到人脸、证件、屏幕聊天内容、危险物品时，不生成学习卡。
- 如果图片无法确定，返回引导重拍。

### 9.2 故事生成 Prompt 要求

输入：

- 来源单词
- 年龄段
- 绘本页数
- 插画风格

输出 JSON：

```json
{
  "title": "Ben and the Apple",
  "pages": [
    {
      "pageIndex": 1,
      "sentence": "Ben sees an apple.",
      "focusWords": ["apple"],
      "imagePrompt": "A soft 3D children's picture book scene..."
    }
  ]
}
```

规则：

- 每页一句英文。
- 不使用复杂从句。
- 不出现惊吓、惩罚、危险尝试。
- 毛毡元素应写成温暖手作装饰，例如羊毛贴片、缝线边框、布艺云朵、纽扣星星、柔软小包和贴布路径。

### 9.3 插图生成要求

基础风格：

- 软萌 3D 儿童绘本
- 圆润角色
- 暖黄色学习感
- 淡彩天空、云朵、星星
- 轻量毛毡/贴布/缝线/羊毛纹理装饰

禁止：

- 尖锐、脏污、破损或惊悚化的手作材质
- 血迹、惊悚表情、阴暗恐怖场景
- 写实儿童人脸
- 过多文字嵌入图片

## 11. 技术方案

### 11.1 iOS 客户端

建议技术栈：

- SwiftUI：界面与导航
- AVFoundation：相机与音频
- AVSpeechSynthesizer：英文朗读 MVP
- SwiftData：本地单词本和绘本数据
- PhotosUI：相册选择
- URLSession：后端 API 调用

客户端模块：

- AppShell：TabView 和全局路由
- CameraFeature：拍照、相册选择、识别状态
- WordFeature：单词学习卡
- StoryFeature：绘本生成、阅读器、自动播放
- WordbookFeature：单词本
- ParentFeature：家长设置
- SharedServices：识别服务、故事服务、朗读服务、存储服务

### 11.2 后端

后端职责：

- 接收客户端上传图片。
- 调用视觉识别模型。
- 过滤不适合儿童的结果。
- 生成故事结构。
- 生成插图。
- 返回结构化数据。
- 控制 API key，不暴露到 iOS 客户端。

建议接口：

- `POST /v1/recognize-object`
- `POST /v1/storybooks`
- `GET /v1/storybooks/{id}`
- `POST /v1/safety/check`

### 11.3 存储策略

MVP 默认：

- 原始照片不长期保存。
- 本地保存压缩缩略图。
- 绘本插图保存本地缓存。
- 用户可在家长中心删除全部本地数据。

## 12. 隐私与儿童安全

原则：

- 默认最小化数据收集。
- 默认不公开、不社交、不分享。
- 不做儿童开放输入框。
- 不显示模型自由生成的长文本。
- 允许家长删除全部数据。

需要进一步合规确认：

- 面向美国儿童可能涉及 COPPA。
- 面向中国儿童可能涉及个人信息保护与未成年人网络保护要求。
- 上线前应由专业法务检查隐私政策、家长同意和数据处理流程。

## 13. 商业化方向

MVP 不强推付费。后续可考虑：

- 每日免费生成 3 本绘本。
- 订阅解锁更多绘本风格和更多生成次数。
- 家庭会员支持多个孩子档案。
- 离线词库包。

不建议：

- 儿童侧弹出付费引导。
- 抽卡式付费。
- 排行榜刺激。

## 15. 风险与对策

### 15.1 识别错误

风险：儿童学习错误单词。  
对策：低置信度必须确认；提供备选词；家长可纠正。

### 15.2 生成内容不稳定

风险：故事或图片不适合儿童。  
对策：固定 JSON 结构；安全审核；限制主题；失败时使用模板故事。

### 15.3 成本过高

风险：图片生成和视觉识别成本高。  
对策：缓存；限制每日生成；MVP 使用自动播放绘本代替真实视频生成。

### 15.4 隐私风险

风险：儿童照片上传引发家长顾虑。  
对策：明确说明；默认不保存原图；可关闭云端识别；后续探索端侧识别。

## 16. 验收清单

- 儿童可以在 3 步内完成拍照识词。
- 识别结果页可以播放英文发音。
- 单词可以保存到单词本。
- 一个单词可以生成 4 页英文绘本。
- 绘本可以自动播放并朗读。
- 家长可以删除本地数据。
- 所有儿童侧错误反馈都温和、不挫败。
- 毛毡元素表达为温暖手作装饰，不产生廉价或杂乱感。
